vue页面的框架结构

template作为设计师给你的h5模板，理论上你应该放在这个里面，但是却不一定要全部放在里面，实际操作上我们可以把需要变动的那部分抠出来放进去。

script毫无疑问就是我们主要的代码部分，如果要和template发生点什么关系，我们可以简单的理解为，template里面预备了一些坑位，而坑位里面需要一些数据，就是从script里面生成出来的，因为，我们进一步理解，script里面的一些属性啊，计算属性啊，方法啊，其实全部都是为了数据，数据，数据，包括后面要讲到的vuex，本质上也是处理数据。数据变了，template就跟着变了，嗯，这既是MVVM了。

style作为设计师给到你的css，你可以放在这里面，当然，我了解到有些团队的设计师并不会给一块块的css，那么实际上你也没有必要放到这里了，当然，这改变不了这里是放样式的这样一个事实。

uni-app使用Vuex指南

Posted on 2019-06-04 Edited on 2019-12-10 In 小程序 Views: Valine:

今天遇到这样一个需求，需要将上一个页面的一坨数据带到下一个页面去，不只打怎么办才好。

大概查了一下，有四中方式；

  1. 公用模块

    这种方式就是做一个单例，把需要的变量放在一个js文件中，然后到处引用，可想而知有多混乱。

  2. 挂载 Vue.prototype

    在 main.js 中挂载属性/方法，注意坑websiteUrl，这个在其他地方改了，另外的文件读到的还是后面初始化的这个值。

  3. globalData

    在小程序中可以在 App 上声明 globalData，但在 Vue 中没有这个属性，可以使用 API 读写这个值。

    赋值：getApp().globalData.text = 'test'

    取值：console.log(getApp().globalData.text) // 'test'

    1. Vuex

      Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

      虽然说很好，但是我玩的时候还是爬了点坑。

图解机器学习笔记

Posted on 2019-05-27 In 人工智能 Views: Valine:

回归

简单直观的了解就是曲线的拟合，可以用来预测房价，给定输入，预测输出。

分类

主要是做模式识别，比如，判断一幅画是一条狗，还是一只猫，还是一束花。

聚类

同样可以通过学习，识别出类别，但是不同于分类的是，这种是无监督学习，因为学习的样本事先也不知道它具体属于那种类别。

最小二乘

最小二乘的原理是是的模型函数与真实值之前的这个差值得均方差最小的一种最优化方法

梯度下降

梯度下降过程中极容易出现拿到的是局部最优解，好像有一种方式可以有效的避免求得到的是局部最优解，那就是步伐先搞大点，然后在搞小点。

万万没想到react请求数据花样如此之多

接触react一个星期，也慢慢熟悉了一些概念，比如HOC(高阶组件)、jsx、函数式组件、HOOK，感觉react也没有别人说的学习曲线陡峭，难上手等等，给我的感觉，如果你会Vue，上手React真的会非常快，不要被这些概念给吓到，这样的一些概念的出现，一定是有着他的道理的，无外乎包含但不限于以下两点理由：

  1. 为了代码复用，比如HOC，自定义HOOK

  2. 为了代码更加简洁，更加好理解，比如jsx,函数式组件。

那么，我们所理解的React的模式，其实归根结底就是UI=Render(State),这其实和Vue乃至整个前端的哲学并无任何冲突，相反，是一个统一。说来说去，说简单点，一个web应用，应该是状态驱动的，而状态=数据+逻辑所以，我们的UI=Render(Data+Logic)

所以来看看怎么做Transfer learning，也就是所谓的迁移学习，那么什么是迁移学习呢？迁移学习(Transfer learning) 顾名思义就是就是把已学训练好的模型参数迁移到新的模型来帮助新模型训练。考虑到大部分数据或任务是存在相关性的，所以通过迁移学习我们可以将已经学到的模型参数（也可理解为模型学到的知识）通过某种方式来分享给新模型从而加快并优化模型的学习效率不用像大多数网络那样从零学习。

tensorflow入门初探

Posted on 2019-05-20 In 人工智能 Views: Valine:

tensorflow lite是干嘛的

TensorFlow Lite is an open source deep learning framework for on-device inference

上面这句话是官网上面的原话，直接了当的说明了TensorFlow Lite是一个可以部署在端设备上的深度学习框架，这说明了，深度学习已经有着覆盖移动端设备的趋势了，所以，移动开发者似乎又有了新的学习任务。

那么，目前TensorFlow Lite能干些啥

  1. 图片分类

  2. 物体监测

  3. 聊天机器人

嗯，看起来似乎有点意思，那呢他是如何工作的呢？

大抵就是经过上面四个步骤

  1. 选择一个已有模型改吧改吧，当然你牛你可以自己训练，大部分都是选择开源模型

  2. 将它转换为lite可用的模型，这一步就是一个函数调用，不怕

  3. 部署到端设备，相当于加载一个数据库那么easy

  4. 进行识别调优，准确率需要自己不断地训练去教育它。
